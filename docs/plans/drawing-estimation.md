# 계획서 — 도면 정밀검토(불일치 발견)·적산 고도화

> 작성: 2026-07-18 · 상태: 계획 확정 대기 → 구현은 M6b(검토)·M6c(적산)
> 목적: ① 입면도·평면도·인테리어도면·창호일람표 간 불일치를 체계적으로 발견·기록하고
> ② 도면에서 면적·물량을 정밀 산출해 적산·견적(요율 적용)까지 잇는다.

## 1. 도면 간 불일치 발견 계획

### 1-1. 원리: "무엇과 무엇을 대조하는가"를 표준화

불일치는 우연히 찾는 게 아니라 **대조 매트릭스**를 순서대로 확인해 찾는다.

| # | 대조 쌍 | 확인 항목 | 흔한 불일치 사례 |
|---|---|---|---|
| C1 | 창호일람표 ↔ 평면도 | 기호(W1, D1…)별 수량·위치·크기 | 일람표 수량 ≠ 평면 배치 수량, 기호 누락 |
| C2 | 창호일람표 ↔ 입면도 | 입면에 표현된 창호 형상·분할·개폐방식 | 입면 창 분할과 일람표 상세 불일치 |
| C3 | 평면도 ↔ 입면도 | 개구부 위치·폭, 층고·파라펫 높이, 레벨(FL/SL) | 평면 개구부가 입면에 없음, 층고 상이 |
| C4 | 평면도 ↔ 인테리어(전개도) | 실 구획·문 위치·마감 범위 | 전개도 벽 길이 ≠ 평면 치수 |
| C5 | 실내재료마감표 ↔ 인테리어도면 | 실별 바닥/벽/천장 마감재 | 마감표와 도면 표기 상이 |
| C6 | 치수 자체 검산 | 부분 치수 합 = 전체 치수, 면적표 = 실측정 | 치수 합 불일치, 면적표 오기 |
| C7 | 구조 ↔ 건축 평면 (확장) | 기둥·벽 위치, 개구부 | 구조 개구부 누락 |
| C8 | 차수 간 (Rev.) | 변경 도면 반영 범위 | 변경이 일부 도면에만 반영 |

### 1-2. 앱 워크플로 (검토 세션)

```
① 도면 세트 등록 — 유형 태그 확장: 평면도/입면도/인테리어/창호일람표/마감표/구조/설비 + 차수
② 검토 세션 생성 — 현장 선택 → 대조 매트릭스(C1~C8)가 체크리스트로 자동 생성
③ 항목별 검토 — 화면 분할 뷰(도면 A | 도면 B 나란히) + 기존 오버레이 비교(차수 간)
   · OCR 보조: 창호일람표에서 기호·수량 추출 → 평면 카운트 입력과 자동 대조표
④ 불일치 기록 — 도면 위 클릭으로 위치 핀 + 유형·심각도·설명 입력 (해당 부위 캡처 자동 저장)
⑤ 질의서 출력 — 불일치 목록을 "설계 질의서(RFI)" A4 양식으로 인쇄/XLSX → 설계사·발주처 발송
   질의 회신 상태 추적: 발견 → 질의 → 회신 → 반영확인
```

**UI/UX 요점**
- 분할 뷰: 좌우 두 뷰어 독립 페이지·줌 + "동기 스크롤" 토글.
- 위치 핀: 기존 측정 오버레이와 동일한 좌표 체계(기준배율 px) 재사용 — measure.js 그대로 활용.
- 심각도 3단계: 경미(표기)/중요(수량·치수)/치명(시공 불가) — 색 배지.
- 질의서(RFI)는 문서대장에 "발신"으로 자동 등록 → 회신기한 관리에 합류 (기존 흐름 재사용).

### 1-3. 데이터 모델 추가

```

Discrepancy: { sessionId, checkCode(C1~C8), drawingAId, drawingBId,

```

### 1-4. 자동화 로드맵 (정직한 단계 설정)

| 단계 | 자동화 수준 |
|---|---|
| M6b (지금) | 대조 매트릭스 워크플로 + 분할 뷰 + OCR 기호 추출 대조표 = **사람이 찾고 앱이 안내·기록** |

## 2. 적산(면적 산출) 정밀화 계획

### 2-1. 도면 읽기 표준 절차 (측정 신뢰도 확보)

1. **축척 이중 확인**: 도면 표제란의 명기 축척(예: 1/100) 입력 + 치수선 2점 보정 → 두 값의 오차가 2% 초과 시 경고 (스캔 왜곡·비율 오류 감지).
2. **페이지별 축척**: 한 PDF에 여러 축척 혼재(평면 1/100, 상세 1/30) → 축척을 도면 단위가 아닌 **페이지 단위**로 저장하도록 개선.
3. **측정 표준 순서**: 실(Room) 경계 → 개구부 → 특이 형상. 같은 실은 "바닥면적 + 둘레 + 층고" 세트로 측정.

### 2-2. 실(Room) 단위 구조화 — 측정을 물량으로

현재: 개별 측정값 나열 → 개선: **실 단위로 묶어 파생 물량 자동 계산**

```
Room: { siteId, drawingId, name(101호 거실), floorArea(측정), perimeter(측정),
  height(입력, 층고), openings[{ windowCode(창호일람표 연동), w, h, count }] }

파생 물량 (자동):
  바닥 마감 = floorArea
  천장 마감 = floorArea
  벽 마감   = perimeter × height − Σ(개구부 w×h)
  걸레받이  = perimeter − Σ(문 폭)
```

- 창호일람표 OCR 결과를 `WindowSchedule`로 저장 → 개구부 입력 시 기호 선택만으로 크기 자동.
- 실별 집계표 = **물량산출서(BOQ) 초안**: 실 × 부위 × 수량 → XLSX.

### 2-3. 적산 요율·원가 요율 체계

> 요율은 연도·발주처별 고시로 바뀜 → **하드코딩 금지, 설정 가능한 요율 마스터(RateSet)**로 관리.

```
RateSet(요율 마스터): { name(2026 표준), validFrom,
  rates: { 간접노무비율, 산재보험료율, 고용보험료율, 건강연금요율,
           산업안전보건관리비율, 환경보전비율, 기타경비율, 일반관리비율, 이윤율, 부가세율 } }

UnitPrice(단가): { workCode(바닥-강마루, 벽-수성페인트…), unit(㎡/m/EA), materialCost, laborCost, expense }
```

**견적 계산 흐름 (원가계산서 구조)**
```
수량(Room 파생 물량) × 단가(UnitPrice) = 직접공사비(재료비+노무비+경비)
→ 간접노무비 = 직접노무비 × 요율
→ 보험료·안전관리비·환경보전비 등 = 각 기준액 × 요율
→ 일반관리비 = (재료비+노무비+경비) × 요율
→ 이윤 = (노무비+경비+일반관리비) × 요율
→ 합계 + 부가세 = 견적금액
```

- 산출 근거 추적: 견적서의 모든 숫자는 "어느 실, 어느 측정, 어느 요율"에서 왔는지 클릭으로 역추적 가능하게 (신뢰성 = 상급자 설득력).
- 초기 단가는 사용자가 입력/XLSX 가져오기. (물가정보지·일위대가 연동은 장기 과제)

### 2-4. UI/UX 계획 (견적서 작성 메뉴 실기능화)

```
estimate.html (견적·적산)
┌ [현장 ▼] [도면 ▼]                                 [물량산출서 XLSX] [견적서 인쇄] ┐
│ 탭: [① 실 측정] [② 물량산출서] [③ 단가·요율] [④ 견적서]                        │
│ ① 실 측정: 도면 뷰어(기존) + 우측 실 목록(실명·바닥·둘레·층고·개구부) — 측정 후 실에 귀속 │
│ ② 물량산출서: 실×부위 자동 물량 표, 수동 보정 열, 산출 근거 표시                   │
│ ③ 단가·요율: UnitPrice 표 편집, RateSet 선택·편집                                 │
│ ④ 견적서: 원가계산서 구조 A4 (직접비→간접비→일반관리비→이윤→부가세), 결재란        │
└──────────────────────────────────────────────────────────────────────────────┘
```

## 4. 검증 기준

- 축척 이중 확인: 명기 축척 vs 2점 보정 오차 2% 초과 시 경고 동작
- 알려진 도면(치수 명확)으로 실 1개 측정 → 벽 마감 물량 수기 계산과 일치
- C1 대조: 일람표 수량 vs 평면 카운트 불일치 시나리오 재현 → 불일치 기록 → RFI 인쇄
- 견적서: 샘플 단가·요율로 원가계산서 합계 검산 (엑셀 수기 계산과 대조)
- 질의서가 문서대장 "발신"으로 등록되고 회신기한이 캘린더에 표시
