# 계획서 — 착공·준공·키스콘 서류 체크리스트 모듈

> 작성: 2026-07-18 · 상태: 계획 확정 대기 → 구현은 M6a
> 목적: 시·군청별 착공/준공 서류와 하도급 키스콘(KISCON) 등록 조건을
> 현장별 체크리스트로 관리해 누락·기한 초과를 없앤다.

## 1. 관리 대상 정의

### 1-1. 하도급 키스콘 등록 관련 조건 (건설산업기본법 기준)

> ※ 금액 기준·요율은 법령 개정으로 바뀔 수 있으므로, 아래는 **초기 탑재값**이며
> 템플릿 편집 화면에서 수정 가능해야 한다. 적용 전 최신 법령 확인 필수.

**하도급 계약 전 점검 조건 (계약 가능 여부)**
| 조건 | 내용 |
|---|---|
| 업종 보유 | 하수급인이 해당 공종의 건설업 등록(종합/전문) 보유 |
| 일괄하도급 금지 | 도급받은 공사 전부 또는 주요 부분의 일괄 하도급 금지 |

## 2. 수집 계획 (시·군청별 서식·요건이 다른 문제)

1. **표준 템플릿 내장**: 위 1-2, 1-3을 기본 템플릿으로 앱에 탑재.
2. **현장 개설 시 현지화**: 현장 등록 → 해당 시·군청 건축(허가)과에 체크리스트·서식 확인
   (홈페이지 고시/전화) → 표준 템플릿을 복제해 "OO시청 2026-07 확인본"으로 저장.
   템플릿에 `발행처·확인일·확인자` 메타를 기록해 신뢰도 관리.
3. **서식 파일 보관**: 서식 PDF/HWP는 도면 모듈과 동일하게 IndexedDB(files.js) 저장,
   체크리스트 항목에서 연결. (연말 Supabase Storage 이관)
4. **버전 관리**: 지자체 요건 변경 발견 시 템플릿 새 버전 생성(기존 현장 체크리스트는 영향 없음).
5. **축적**: 현장을 거칠수록 지자체별 템플릿 라이브러리가 쌓임 → 공사부 공유 자산 (연말 멀티유저의 핵심 가치).

## 3. 데이터 모델 추가 (DATA_SCHEMA.md에 반영 예정, schemaVersion 2)

```
ChecklistTemplate: { name, kind(착공/준공/하도급), authority(지자체명 or "표준"),
  confirmedAt, confirmedBy, version,
  items: [{ title, group, agency, requiredWhen(항상/대상공사/해당시),
            formNote(서식 출처), defaultDueRule(예: 계약+30일) }] }

SiteChecklist: { siteId, templateId, kind, startedAt,
  items: [{ templateItemIdx, status(해당없음/준비중/제출/보완요청/완료),
            dueDate, submittedAt, documentId(문서대장 연결), fileId(서식·제출본), ownerId, memo }] }

Subcontract(하도급): { siteId, company, trade(공종), amount, contractDate,
  licenseChecked, bondIssued(지급보증), kisconNotifiedAt, kisconDueDate(계약+30일 자동),
  status(검토중/계약/통보완료/변경통보필요), memo }
```

- 기한 있는 항목(`dueDate`, `kisconDueDate`)은 **캘린더 합성 소스에 추가** (기존 회신기한과 동일 패턴).
- 대시보드 KPI에 "체크리스트 미완료·기한 임박" 추가.

## 4. UI/UX 계획

### 4-1. 내비게이션
- 문서·계약 그룹의 준비중 메뉴를 실기능화:
  - **착공·준공 체크리스트** (`checklists.html`)
  - **하도급·키스콘** (`subcontracts.html`)

### 4-2. 체크리스트 화면 (checklists.html) — 와이어프레임

```
┌ 상단바: [현장 선택 ▼] [착공|준공 탭]           [+ 템플릿에서 시작] [인쇄] ┐
│ ┌ 진행 요약 카드 ──────────────────────────────────────────┐          │
│ │  제출 18 / 24  ▓▓▓▓▓▓▓▓░░ 75%   기한 임박 2건  보완요청 1건 │          │
│ └──────────────────────────────────────────────────────────┘          │
│ [그룹: 전체|인허가|안전·환경|계약·인력|현장준비]  [상태 필터] [검색]      │
│ ┌ 항목 테이블 ─────────────────────────────────────────────┐          │
│ │ 상태▾      항목            관할     기한(D-day)  문서  담당  비고 │      │
│ │ [제출]     착공신고서       세움터   07/24 D-6   📎    이대리      │      │
│ │ [보완요청] 안전관리계획서   국토안전  07/22 D-4   📎    이대리  ⚠  │      │
│ │ [준비중]   비산먼지 신고    환경과    -           -     -          │      │
│ └──────────────────────────────────────────────────────────┘          │
│ 행 클릭 → 우측 슬라이드 패널: 서식 출처·파일, 문서대장 연결, 메모, 이력   │
└───────────────────────────────────────────────────────────────────────┘
```

**UX 원칙**
1. 상태는 셀렉트 배지 5단계(해당없음=회색/준비중=파랑/제출=남색/보완요청=주황/완료=초록) — 행에서 바로 변경, 클릭 2번 이내.
2. 진행률은 "해당없음 제외" 분모로 계산 (실제 체감 진행률).
3. 기한 지난 항목은 행 강조(기존 `overdue` 패턴)·대시보드 노출.
4. 인쇄 = 제출 대장 형식 A4(결재란 포함) — 발주처·상급자 보고용.
5. 항목→문서대장 양방향 연결: 제출 공문을 대장에 등록하면 체크리스트 상태 자동 "제출" 제안.

### 4-3. 하도급·키스콘 화면 (subcontracts.html)

- 하도급 계약 카드 목록: 업체·공종·금액·상태(검토중/계약/통보완료).
- **등록 위저드(3단계)**: ① 업체·공종·금액 입력 → ② 조건 자동 점검 표
  (업종 보유? 지급보증? 적정성 심사 대상?) 체크 → ③ 통보기한 D-30 자동 계산 → 캘린더 이벤트 생성.
- 상단 경고 스트립: "키스콘 통보기한 임박 N건".

### 4-4. 템플릿 관리 (settings.html 확장)
- 템플릿 목록(표준/지자체별) → 복제·편집(항목 추가/삭제/그룹 변경) → 확인일·발행처 기록.

## 5. 구현 순서 (M6a — 3단계)

1. **schemaVersion 2 마이그레이션** + 표준 템플릿 시드 + checklists.html (조회·상태 변경·진행률·인쇄)
2. 캘린더·대시보드 연동 + 문서대장 연결 + 템플릿 관리
3. subcontracts.html 위저드 + 키스콘 기한 자동화

## 6. 검증 기준

- 표준 템플릿으로 현장 1곳 착공 체크리스트 생성 → 상태 변경 → 새로고침 유지 → 진행률 정확
- 기한 입력 항목이 캘린더·대시보드에 나타남
- 하도급 등록 → 통보기한 D-30 이벤트 자동 생성
- 인쇄 A4 1~2장, 결재란 포함
- domain-reviewer 에이전트로 용어·법령 표기 검수
